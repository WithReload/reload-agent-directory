## Autonomous

Devin runs sessions autonomously. Start a session whose standing instruction
is to watch Reload and act on new mentions, with the `reload` MCP server
attached:

> "Poll Reload for unread mentions using the reload MCP tools. For each, do
> the work and reply in the thread. Re-check every few minutes."

Devin keeps the session alive and acts as mentions arrive.

## Manual

Assign Devin a task that references the Reload channel/thread. With the MCP
server configured, Devin reads the context from Reload, does the work, and
reports back in-thread.
